[Keystatic Cloud](https://keystatic.cloud) simplifies authentication with GitHub for your Keystatic projects. Instead of setting up a custom GitHub app and managing environment variables yourself, Keystatic Cloud handles the OAuth flow for you.

Editors can log in to the admin UI and edit content without needing a GitHub account of their own. Keystatic Cloud authenticates with GitHub on their behalf.

Keystatic Cloud also provides opt-in products:

- **Cloud Images** — image upload, storage, optimisation, and delivery
- **Multiplayer editing** (experimental) — real-time collaboration on the same document

## Authentication

Keystatic Cloud connects to your GitHub repository and manages authentication for one or multiple Keystatic projects. It skips the more involved process of [setting up GitHub mode manually](/concepts/github-mode), while also letting team members edit content without needing individual GitHub accounts.

## Teams and projects

Keystatic Cloud organises projects into teams. Each team can have multiple projects, and each project is connected to a specific GitHub repository.

User access is set at the team level — every user in a team has access to all projects within that team.

## Free vs Pro plans

Creating a [Keystatic Cloud account](https://keystatic.cloud) is free. You can create as many teams and projects as you need.

| | Free | Pro |
|---|---|---|
| Teams | Unlimited | Unlimited |
| Projects per team | Unlimited | Unlimited |
| Users per team | Up to 3 | Unlimited |
| Cloud Images | — | ✓ (opt-in) |
| Multiplayer editing | — | ✓ (experimental) |
| Price | Free | $10/month base + $5/month per user beyond 3 |

<Note>
  A Pro plan only applies to the specific team it's enabled on. Other teams in your account remain on the free plan unless you upgrade them separately.
</Note>

To upgrade a team, go to the **Billing** tab for that team in Keystatic Cloud.

## Pro features

Besides raising the user limit beyond 3, Keystatic Cloud Pro unlocks:

- **Multiplayer editing** (experimental): collaborate with other editors in real-time on the same document.
- **Cloud Images**: upload, transform, and serve optimised images without committing them to your GitHub repository.

Once subscribed to Pro, you can enable these features from the **Settings** tab of the relevant team.

## Cloud Images

Cloud Images is an opt-in image storage and delivery service that optimises your images for the web.

Each Keystatic Cloud project has its own Image Library where you can upload images and copy URLs to use in your content. Images are served from Keystatic's CDN and can be transformed on the fly using URL query parameters.

### Image optimisation via URL query parameters

You can append query parameters to any Keystatic Cloud image URL to resize and re-encode the image on delivery:

| Parameter | Shorthand | Description |
|---|---|---|
| `fit` | — | How the image is resized. Values: `scale-down`, `contain`, `cover`, `crop` |
| `format` | `f` | Output format. Values: `png`, `avif`, `webp`, `jpeg`. If omitted, Keystatic Cloud auto-detects the best format for the browser. |
| `quality` | `q` | Image quality. A number between `1` and `100`. |
| `height` | `h` | Output height in pixels. A number between `1` and `10240`. |
| `width` | `w` | Output width in pixels. A number between `1` and `10240`. |

You can provide one or both of `height` and `width`. The effect depends on the `fit` value.

Example URL with transform parameters:

```
https://[IMAGE_URL]?width=240&height=480&fit=crop
```

### Cloud image field

Keystatic provides a `cloudImage` field that you can use in place of the regular [`image` field](/fields/image) when:

- Keystatic is configured with `storage: { kind: 'cloud' }`, and
- The Image Library is enabled for the project in Keystatic Cloud.

```ts keystatic.config.ts
import { config, collection, fields } from '@keystatic/core'

export default config({
  storage: {
    kind: 'cloud',
  },
  cloud: {
    project: 'your-team/your-project',
  },
  collections: {
    posts: collection({
      label: 'Posts',
      slugField: 'title',
      schema: {
        title: fields.slug({ name: { label: 'Title' } }),
        cover: fields.cloudImage({
          label: 'Cover image',
        }),
      },
    }),
  },
})
```

### Cloud image component block (experimental)

Keystatic also provides a `CloudImage` component block for use inside a document field. Like the field, it requires cloud mode and the Image Library to be enabled for the project.

<Note>
  Documentation for the `CloudImage` component block is coming soon.
</Note>

## Configuring your project

To use Keystatic Cloud, set `storage.kind` to `'cloud'` and add the `cloud.project` property in your `keystatic.config.ts`. The project value is the team name and project name from your Keystatic Cloud account, separated by a slash.

```ts keystatic.config.ts
import { config } from '@keystatic/core'

export default config({
  storage: {
    kind: 'cloud',
  },
  cloud: {
    project: '[TEAM_NAME]/[PROJECT_NAME]',
  },
  // collections and singletons ...
})
```

<Tip>
  Each project in Keystatic Cloud has a **Settings** page with a ready-to-paste code snippet for your config file.
</Tip>

## Getting started with Keystatic Cloud

<Steps>
  <Step title="Create a Keystatic Cloud account">
    Go to [keystatic.cloud](https://keystatic.cloud) and sign up for a free account.
  </Step>

  <Step title="Create a team and project">
    Create a new team, then add a project inside it. Connect the project to your GitHub repository.
  </Step>

  <Step title="Copy the config snippet">
    Open the project settings page in Keystatic Cloud and copy the configuration snippet.
  </Step>

  <Step title="Update your keystatic.config.ts">
    Paste the snippet into your `keystatic.config.ts`, replacing the existing `storage` value:

    ```ts
    storage: {
      kind: 'cloud',
    },
    cloud: {
      project: 'your-team/your-project',
    },
    ```
  </Step>

  <Step title="Deploy your project">
    Deploy your project. Editors can now log in via Keystatic Cloud without needing individual GitHub accounts.
  </Step>
</Steps>
