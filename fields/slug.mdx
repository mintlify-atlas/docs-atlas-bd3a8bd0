---
title: "Slug field"
description: "Pair a display name with an auto-generated URL-friendly slug for use as a collection entry identifier."
---

The `slug` field stores a text string (such as a post title or an author name) alongside an auto-generated URL-friendly slug. Editors see two inputs in the Admin UI: one for the display name and one for the slug, which can be manually overridden.

This field is used in conjunction with the `slugField` option on a collection to make the slug serve as the file/folder name for each entry.

## Usage

```typescript
import { fields } from '@keystatic/core'

title: fields.slug({
  name: {
    label: 'Title',
    description: 'The title of the post.',
  },
  // Optional slug field customisation
  slug: {
    label: 'SEO-friendly slug',
    description: 'This will define the file/folder name for this entry.',
  },
})
```

## Return type

Returns `{ name: string; slug: string }` when reading as a regular schema field. When used as the `slugField` on a collection, the Reader API returns only the `name` string (the slug becomes the entry key).

## Example

```typescript
import { collection, config, fields } from '@keystatic/core'

export default config({
  collections: {
    posts: collection({
      label: 'Posts',
      slugField: 'title',
      schema: {
        title: fields.slug({
          name: {
            label: 'Title',
            validation: { isRequired: true },
          },
          slug: {
            label: 'URL slug',
            description: 'The URL path segment for this post.',
          },
        }),
      },
    }),
  },
})
```

<Note>
  The `slug` field can also be used inside an [`array`](/fields/array) field with the `slugField` option. This replaces numeric indexes in file paths for nested documents or images.
</Note>
