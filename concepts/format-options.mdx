---
title: "Format options"
description: "Configure Keystatic's output file format for collections and singletons — choose between YAML, JSON, or Markdoc with the format option."
---

Keystatic can store data in multiple formats: YAML, JSON, Markdoc (`.mdoc`), and MDX. The `format` option on a collection or singleton controls which format is used.

By default, entries are stored in a YAML file. If the collection schema contains a `document`, `markdoc`, or `mdx` field, a separate `.mdoc` or `.mdx` file is generated for that field's content.

<Note>
  The name and directory structure of entries is also affected by the `path` setting. See [Content organisation](/concepts/content-organisation) for details.
</Note>

---

## Default format (YAML)

With no `format` option specified, a collection with a `markdoc` field generates two files per entry:

```typescript
blog: collection({
  label: 'Blog',
  slugField: 'title',
  schema: {
    title: fields.slug({ name: { label: 'Title' } }),
    publishedDate: fields.date({ label: 'Published date' }),
    body: fields.markdoc({ label: 'Body' }),
  },
})
```

Creating a `what-a-day` entry produces:

```bash
blog/what-a-day
├── index.yaml
└── body.mdoc
```

<CodeGroup>

```yaml index.yaml
title: What a day
publishedDate: 2023-07-27
```

```markdown body.mdoc
What a **beautiful** day!

## Let's go to the beach

I say we pack our swimmers and towels and head to the beach.

Who's with me?
```

</CodeGroup>

## JSON data format

Use `format: { data: 'json' }` to store metadata as JSON instead of YAML:

```typescript
blog: collection({
  label: 'Blog',
  slugField: 'title',
  format: { data: 'json' },
  schema: { /* ... */ },
})
```

This produces an `index.json` file instead of `index.yaml`:

```bash
blog/what-a-day
├── index.json
└── body.mdoc
```

```json
{
  "title": "What a day",
  "publishedDate": "2023-07-27"
}
```

## Single-file output with `contentField`

The `format.contentField` option combines all fields into a single file, with metadata in frontmatter and the document content below it.

Set `contentField` to the key of a `markdoc`, `mdx`, or `document` field in your schema:

```typescript
blog: collection({
  label: 'Blog',
  slugField: 'title',
  format: { contentField: 'body' },
  schema: {
    title: fields.slug({ name: { label: 'Title' } }),
    publishedDate: fields.date({ label: 'Published date' }),
    body: fields.markdoc({ label: 'Body' }),
  },
})
```

Instead of two separate files, Keystatic now outputs a single `index.mdoc` file:

```bash
blog/what-a-day
└── index.mdoc
```

```markdown
---
title: What a day
publishedDate: 2023-07-27
---

What a **beautiful** day!

## Let's go to the beach

I say we pack our swimmers and towels and head to the beach.

Who's with me?
```

<Note>
  Setting `format.contentField` is also required for the `entryLayout: 'content'` option to take effect. See [Entry layout](/concepts/entry-layout).
</Note>

## Type signature

```typescript
type DataFormat = 'json' | 'yaml';

type Format =
  | DataFormat
  | {
      data?: DataFormat;
      contentField?: string | [string, ...string[]];
    };
```
