## What is Keystatic?

Keystatic is a Git-based CMS that gives content editors a beautiful admin UI while keeping all your content in Markdown, YAML, or JSON files — directly in your repository. No database, no external service, no data migrations.

Built with TypeScript from the ground up, Keystatic integrates natively with **Next.js**, **Astro**, and **Remix**. Define your content schema once using the TypeScript API, and get full type safety throughout your project.

## Get started in minutes

<Steps>
  <Step title="Scaffold a new project">
    Use the `create-keystatic` CLI to scaffold a new project with Keystatic pre-configured:

    <CodeGroup>
    ```bash npm
    npm create keystatic@latest
    ```
    ```bash pnpm
    pnpm create keystatic@latest
    ```
    ```bash yarn
    yarn create keystatic
    ```
    </CodeGroup>
  </Step>
  <Step title="Or add to an existing project">
    Install Keystatic into your existing Next.js, Astro, or Remix project and follow the integration guide for your framework.

    <CardGroup cols={3}>
      <Card title="Next.js guide" icon="n" href="/installation/next-js" />
      <Card title="Astro guide" icon="star" href="/installation/astro" />
      <Card title="Remix guide" icon="zap" href="/installation/remix" />
    </CardGroup>
  </Step>
  <Step title="Define your content schema">
    Create a `keystatic.config.ts` and define collections and singletons using the TypeScript API:

    ```typescript keystatic.config.ts
    import { config, collection, fields } from '@keystatic/core'

    export default config({
      storage: { kind: 'local' },
      collections: {
        posts: collection({
          label: 'Posts',
          slugField: 'title',
          path: 'src/content/posts/*',
          format: { contentField: 'content' },
          schema: {
            title: fields.slug({ name: { label: 'Title' } }),
            publishedDate: fields.date({ label: 'Published date' }),
            content: fields.markdoc({ label: 'Content' }),
          },
        }),
      },
    })
    ```
  </Step>
  <Step title="Open the admin UI">
    Navigate to `/keystatic` in your browser to start managing content. Changes are saved as files in your repository.
  </Step>
</Steps>

## Reading your content

Use the Reader API to load your content at build time or runtime with full type safety:

```typescript
import { createReader } from '@keystatic/core/reader'
import keystaticConfig from './keystatic.config'

const reader = createReader(process.cwd(), keystaticConfig)

// Read all posts
const posts = await reader.collections.posts.all()

// Read a single post by slug
const post = await reader.collections.posts.read('my-post-slug')
```
