---
title: "Real-time previews with Next.js draft mode"
description: "Set up immediate content previews in Keystatic using Next.js draft mode and the GitHub reader."
---

One downside of Git-based content is the delay between saving a change and seeing it live: you typically have to open a PR and wait for a deploy preview. This recipe shows how to create **immediate previews** of Keystatic content using Next.js [draft mode](https://nextjs.org/docs/app/building-your-application/configuring/draft-mode).

<Note>
  This recipe assumes you have an existing Next.js + Keystatic project that:
  1. Uses the Next.js **App Router**
  2. Uses the **Reader API** to fetch content
  3. Is connected to a GitHub repository (GitHub mode or Keystatic Cloud)
</Note>

---

<Steps>

### Create start and end preview routes

Create `app/preview/start/route.tsx` to enable draft mode when accessed:

```tsx
// app/preview/start/route.tsx
import { redirect } from 'next/navigation';
import { draftMode, cookies } from 'next/headers';

export async function GET(req: Request) {
  const url = new URL(req.url);
  const params = url.searchParams;
  const branch = params.get('branch');
  const to = params.get('to');
  if (!branch || !to) {
    return new Response('Missing branch or to params', { status: 400 });
  }
  draftMode().enable();
  cookies().set('ks-branch', branch);
  const toUrl = new URL(to, url.origin);
  toUrl.protocol = url.protocol;
  toUrl.host = url.host;
  redirect(toUrl.toString());
}
```

Create `app/preview/end/route.tsx` to disable draft mode:

```tsx
// app/preview/end/route.tsx
import { cookies, draftMode } from 'next/headers';

export function POST(req: Request) {
  if (req.headers.get('origin') !== new URL(req.url).origin) {
    return new Response('Invalid origin', { status: 400 });
  }
  const referrer = req.headers.get('Referer');
  if (!referrer) {
    return new Response('Missing Referer', { status: 400 });
  }
  draftMode().disable();
  cookies().delete('ks-branch');
  return Response.redirect(referrer, 303);
}
```

### Add a stop-preview button to your layout

Add a visible indicator and opt-out button to your root layout so editors can exit draft mode:

```tsx
// app/layout.tsx
import { cookies, draftMode } from 'next/headers';

export default async function RootLayout({ children }) {
  const { isEnabled } = draftMode();

  return (
    <div>
      {children}
      {isEnabled && (
        <div>
          Draft mode ({cookies().get('ks-branch')?.value}){' '}
          <form method="POST" action="/preview/end">
            <button>End preview</button>
          </form>
        </div>
      )}
    </div>
  );
}
```

### Add a previewUrl to collections or singletons

In `keystatic.config.ts`, add a `previewUrl` to any collection or singleton you want to preview. Keystatic will render a preview button in the Admin UI that links to the draft-mode URL:

```ts
collections: {
  posts: collection({
    label: 'Posts',
    slugField: 'title',
    path: 'content/posts/*',
    previewUrl: '/preview/start?branch={branch}&to=/posts/{slug}',
    schema: { /* ... */ },
  }),
},
```

The `{branch}` and `{slug}` tokens are substituted automatically by the Admin UI.

### Create a draft-mode-aware reader

Create a utility file that returns the correct reader depending on whether draft mode is active:

```ts
// src/utils/reader.ts
import { createReader } from '@keystatic/core/reader';
import { createGitHubReader } from '@keystatic/core/reader/github';
import keystaticConfig from '../../keystatic.config';
import { cache } from 'react';
import { cookies, draftMode } from 'next/headers';

export const reader = cache(() => {
  let isDraftModeEnabled = false;
  // draftMode() throws outside of request context (e.g. in generateStaticParams)
  try {
    isDraftModeEnabled = draftMode().isEnabled;
  } catch {}

  if (isDraftModeEnabled) {
    const branch = cookies().get('ks-branch')?.value;
    if (branch) {
      return createGitHubReader(keystaticConfig, {
        repo: 'REPO_ORG/REPO_NAME', // replace with your repo
        ref: branch,
        token: cookies().get('keystatic-gh-access-token')?.value,
      });
    }
  }

  return createReader(process.cwd(), keystaticConfig);
});
```

<Warning>
  Replace `'REPO_ORG/REPO_NAME'` with your actual repository. If you are using Keystatic Cloud (not a GitHub App), replace the `token` line with a Personal Access Token.
</Warning>

### Update existing reader usage

Because the new `reader` is a function, update all call sites to invoke it:

```diff
- const posts = await reader.collections.posts.all();
+ const posts = await reader().collections.posts.all();
```

### Test the preview

In the Keystatic Admin UI, create or edit an entry and save it to a new branch. A preview icon will appear next to the Save button. Click it to open the front-end in draft mode, reading live from that branch.

</Steps>
